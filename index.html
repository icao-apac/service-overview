APAC SWIM Service Overview Extensions

This specification defines a set of additional properties that may be used to augment the current set of properties defined in ICAO [[PANS-IM]] for describing SWIM Service Overviews in the APAC region.

Introduction

The ICAO APAC SWIM Task Force has endorsed the proposal at the 5th Meeting to adopt a set of additional properties for describing SWIM Service Overviews. These are aimed at extending the set of properties defined in ICAO [[PANS-IM]].

For a detailed comparison of the properties used in Service Overviews and SWIM Registries, and the original proposal from SWIM T/F5, see [[SWIM5-WP14]].

ICAO PANS-IM Service Overview Properties

The ICAO [[!PANS-IM]] specification defines a set of properties to describe SWIM Service Overviews. This set is required to aid the discoverability of SWIM Information Services and consistent descriptions for Registry interoperability.

The set of Service Overview properties include:

  • Information Service Name
  • Information Service Version
  • Information Service Lifecycle - Status
  • Information Service Lifecycle - Date
  • Information Service Functions
  • Information Category
  • Brief Description of the Information Service
  • Additional Information on the Information Service
  • Quality of the Service
  • Information Service Validation - Type
  • Information Service Validation - Description
  • Filtering Available
  • Access Restrictions
  • Message Exchange Patterns
  • Information Exchange Models
  • Geographical Extent of Information
  • Source of Information
  • Provider Organisation
  • Support Availability
  • Provider Point of Contact

See the ICAO [[!PANS-IM]] specification for a complete definition and usage guide of these properties.

Additional APAC Properties

The additional properties are categorised under:

  • Identification,
  • Discovery, and
  • Technical.

Identification Properties

To support global interoperability of Service Overviews across SWIM Registries, there is requirement for each to be uniquely identified. This field would be a web Uniform Resource Identifier (URI).

Property: Information Service Identifier
Definition: The globally unique identifier for the Service Overview
Notes: The identifier must be a [[!URI]].

Discovery Properties

A key purpose for additional properties for APAC standardisation should be considered in the context of discovery of information services. That is, the information consumer’s need to find services should be the driving requirement. This requirement covers the three Service Overview properties; “Information Category”, “Information Service Functions”, and “Geographical Extend of Information”.

These three properties are free text, and hence, would not provide the capability of a service provider to define explicit structured values (in the form of URI-based vocabualary terms) to enable service consumers to find specific services.

Property: Information Theme
Definition: The primary subject theme of the Service Overview
Notes: The value must be a vocabulary term expressed as a [[!URI]].

Property: Functional Theme
Definition: The primary business function theme of the Service Overview
Notes: The value must be a vocabulary term expressed as a [[!URI]].

To support more advanced Geospatial discovery, three new properties are defined. These properties are reused from the W3C DCAT Profile for ICAO Aeronautical Datasets [[!DCAT-PROFILE]]. This ensures consistency and compatibility with related data standardisation efforts.

Property: Spatial State
Definition: An internationally recognised geographic entity that provides aeronautical information service
Notes: The value must conform to [[!DCAT-PROFILE]].

Property: Spatial Aerodrome
Definition: A defined area on land or water intended to be used either wholly or in part for the arrival, departure and surface movement of aircraft
Notes: The value must conform to [[!DCAT-PROFILE]].

Property: Spatial Airspace
Definition: A defined three dimensional region of space relevant to air traffic
Notes: The value must conform to [[!DCAT-PROFILE]].

Technical Properties

To expand on the more technical descriptive properties, a new property is added to cover the various interface bindings for the SWIM Information Service.

Property: Interface Binding
Definition: A description of an interface binding (including protocols, messages, endpoints, and operations) that enables information services to exchange data with consuming applications
Notes: The value is free text .
